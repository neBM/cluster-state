# Terraform → Kustomize Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Terraform k8s resource management with Kustomize, eliminating the provider API-lag that blocks DRA resource claims.

**Architecture:** Export all live k8s resources to flat YAML manifests grouped by workload, wire a root `kustomization.yaml`, swap the CI pipeline to `kubectl apply -k`, then delete all Terraform files. The ACT_IMAGE build (Task 1) is the long pole — start it first and work the remaining tasks while it builds.

**Tech Stack:** Kustomize, kubectl, Python 3 (PyYAML), GitLab CI

- [ ] **Step 2: Create the export script**

```python
#!/usr/bin/env python3
"""
One-time migration: exports live k8s resources to Kustomize layout.

Usage:
  python3 scripts/export_to_kustomize.py --namespace default --out k8s/
  python3 scripts/export_to_kustomize.py --namespace kube-system --out k8s/
  python3 scripts/export_to_kustomize.py --cluster-scoped --out k8s/
"""

import argparse
import os
import subprocess
import sys
from collections import defaultdict
from pathlib import Path

import yaml

RUNTIME_KEYS = {
    "resourceVersion", "uid", "creationTimestamp", "managedFields", "generation",
}
RUNTIME_ANNOTATIONS = {
    "kubectl.kubernetes.io/last-applied-configuration",
    "deployment.kubernetes.io/revision",
}

NAMESPACED_TYPES = [
    "deployments", "statefulsets", "daemonsets",
    "services", "configmaps", "ingresses",
    "serviceaccounts", "roles", "rolebindings",
    "persistentvolumeclaims",
    "verticalpodautoscalers",
    "externalsecrets",
    "ciliumnetworkpolicies",
]

CLUSTER_TYPES = [
    "storageclasses",
    "clusterroles",
    "clusterrolebindings",
]


def strip(resource: dict) -> dict:
    meta = resource.get("metadata", {})
    for key in RUNTIME_KEYS:
        meta.pop(key, None)
    annotations = meta.get("annotations", {})
    for key in RUNTIME_ANNOTATIONS:
        annotations.pop(key, None)
    if not annotations:
        meta.pop("annotations", None)
    resource.pop("status", None)
    return resource


def app_name(resource: dict) -> str:
    labels = resource.get("metadata", {}).get("labels", {}) or {}
    return (
        labels.get("app.kubernetes.io/name")
        or labels.get("app")
        or labels.get("component")
        or "core"
    )


def kubectl_get(resource_type: str, namespace: str | None) -> list[dict]:
    cmd = ["kubectl", "get", resource_type, "-o", "yaml", "--ignore-not-found=true"]
    if namespace:
        cmd += ["-n", namespace]
    result = subprocess.run(cmd, capture_output=True, text=True)
    if result.returncode != 0:
        print(f"  WARN: {resource_type}: {result.stderr.strip()}", file=sys.stderr)
        return []
    doc = yaml.safe_load(result.stdout)
    if not doc:
        return []
    if doc.get("kind") == "List":
        return [i for i in doc.get("items", []) if i]
    return [doc]


def write_resource(resource: dict, out_dir: Path) -> Path:
    app = app_name(resource)
    kind = resource["kind"].lower()
    name = resource["metadata"]["name"]
    target_dir = out_dir / app
    target_dir.mkdir(parents=True, exist_ok=True)
    filename = target_dir / f"{kind}-{name}.yaml"
    with open(filename, "w") as f:
        yaml.dump(resource, f, default_flow_style=False, allow_unicode=True)
    return filename


def write_kustomization(app_dir: Path) -> None:
    files = sorted(
        p.name for p in app_dir.iterdir()
        if p.suffix == ".yaml" and p.name != "kustomization.yaml"
    )
    kustomization = {
        "apiVersion": "kustomize.config.k8s.io/v1beta1",
        "kind": "Kustomization",
        "resources": files,
    }
    with open(app_dir / "kustomization.yaml", "w") as f:
        yaml.dump(kustomization, f, default_flow_style=False)


def main() -> None:
    parser = argparse.ArgumentParser()
    parser.add_argument("--namespace", help="Namespace to export")
    parser.add_argument("--cluster-scoped", action="store_true")
    parser.add_argument("--out", required=True, help="Output root (e.g. k8s/)")
    args = parser.parse_args()

    out_dir = Path(args.out)
    types = CLUSTER_TYPES if args.cluster_scoped else NAMESPACED_TYPES
    ns = None if args.cluster_scoped else args.namespace

    written: defaultdict[str, list] = defaultdict(list)

    for rtype in types:
        resources = kubectl_get(rtype, ns)
        for res in resources:
            res = strip(res)
            path = write_resource(res, out_dir)
            written[app_name(res)].append(path.name)
            print(f"  {path}")

    for app_dir in out_dir.iterdir():
        if app_dir.is_dir():
            write_kustomization(app_dir)

    print(f"\nSummary: {sum(len(v) for v in written.values())} resources across {len(written)} apps")
    if "core" in written:
        print(f"\nWARN: {len(written['core'])} resources landed in k8s/core/ (no app label) — review manually:")
        for f in written["core"]:
            print(f"  {f}")


if __name__ == "__main__":
    main()
```

## Task 5: Validate against live cluster

This is the critical verification step. A clean diff means the export faithfully captured live state.

- [ ] **Step 1: Run kubectl diff**

```bash
KUBECONFIG=~/.kube/k3s-config kubectl diff -k k8s/ 2>&1 | tee /tmp/kustomize-diff.txt
echo "Exit code: $?"
```

Exit code 0 = no differences. Exit code 1 = differences exist (output shows what would change).

- [ ] **Step 2: Triage any differences**

For each diff block in the output:

- **Runtime-only fields** (e.g. `status`, `generation`, `resourceVersion`): these may appear in resources exported before `strip()` was applied consistently — delete the field from the YAML file.
- **Labels/annotations added by controllers** (e.g. `pod-template-hash`): delete from YAML.
- **Actual content differences**: the live resource was mutated after Terraform last applied — update the YAML to match live state (live state is ground truth).

Re-run `kubectl diff -k k8s/` after each fix until exit code is 0 or all remaining diffs are expected/intentional.

- [ ] **Step 3: Commit any fixes**

```bash
git add k8s/
git commit -m "fix(k8s): resolve kubectl diff issues from export"
```

**Files:**

- [ ] **Step 1: Read current .gitlab-ci.yml to note the kubeconfig before_script block**

```bash

```

- [ ] **Step 2: Replace the entire file**

The new pipeline has two jobs. The kubeconfig `before_script` is shared via a YAML anchor. Replace `.gitlab-ci.yml` with:

```yaml
stages:
  - validate
  - apply

variables:
  ACT_IMAGE: ${CI_REGISTRY}/ben/act:latest

.k8s-base:
  image: ${ACT_IMAGE}
  before_script:
    - mkdir -p ~/.kube
    - |
      cat > ~/.kube/k3s-config << EOF
      apiVersion: v1
      kind: Config
      clusters:
      - name: k3s
        cluster:
          server: ${K8S_HOST}
          certificate-authority-data: ${K8S_CA_CERT}
      contexts:
      - name: default
        context:
          cluster: k3s
          user: ci
      current-context: default
      users:
      - name: ci
        user:
          token: ${K8S_TOKEN}
      EOF
    - chmod 600 ~/.kube/k3s-config
    - export KUBECONFIG=~/.kube/k3s-config

validate:
  extends: .k8s-base
  stage: validate
  script:
    - kustomize build k8s/ > /dev/null
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
    - if: $CI_COMMIT_BRANCH == "main"

apply:
  extends: .k8s-base
  stage: apply
  script:
    - kubectl apply -k k8s/
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
      when: manual
  environment:
    name: production
```
